# Resumo da Execução dos Testes

## Objetivo
Executar os casos de teste do projeto Urban Routes e registrar o status de cada verificação, identificando possíveis falhas na aplicação.

## Tipo de teste
Teste de regressão manual

## Resultado geral
Os testes foram executados com base no roteiro fornecido para o projeto. Durante a validação, foram identificados comportamentos divergentes em relação ao esperado, exigindo abertura de relatórios de bug para acompanhamento e correção.

## Status possíveis utilizados
- Aprovado
- Reprovado
- Ignorado

## Atividades executadas
- Análise dos casos de teste
- Execução das validações no navegador Google Chrome
- Registro dos resultados
- Documentação dos bugs encontrados
- Associação dos bugs aos testes reprovados

## Conclusão
A execução dos testes permitiu validar o comportamento geral do aplicativo e identificar inconsistências que podem impactar a experiência do usuário. O processo reforçou a importância do teste de regressão na garantia da qualidade do software.
